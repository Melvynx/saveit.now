# API Authentication

The SaveIt API uses API keys for authentication. This guide will walk you through generating API keys, making authenticated requests, and following security best practices.

## Getting Your API Key

### Step 1: Generate an API Key

1. **Log in** to your SaveIt dashboard
2. **Navigate** to Settings → API Keys
3. **Click** "Generate New API Key"
4. **Give it a name** (e.g., "Mobile App", "Zapier Integration")
5. **Set permissions** (Read, Write, or Full Access)
6. **Copy the key** - you won't see it again!

### Step 2: Store Securely

**⚠️ Important:** Never commit API keys to version control or expose them in client-side code.

**✅ Secure storage options:**
- Environment variables
- Secure credential managers  
- Encrypted configuration files
- CI/CD secret management systems

## Making Authenticated Requests

### Authorization Header

Include your API key in the `Authorization` header:

```bash
curl -H "Authorization: Bearer sk_live_1234567890abcdef" \
     https://api.saveit.now/v1/bookmarks
```

### Request Examples

#### JavaScript/Node.js
```javascript
const response = await fetch('https://api.saveit.now/v1/bookmarks', {
  headers: {
    'Authorization': 'Bearer sk_live_1234567890abcdef',
    'Content-Type': 'application/json'
  }
});

const bookmarks = await response.json();
```

#### Python
```python
import requests

headers = {
    'Authorization': 'Bearer sk_live_1234567890abcdef',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.saveit.now/v1/bookmarks', headers=headers)
bookmarks = response.json()
```

#### cURL
```bash
curl -X GET \
  -H "Authorization: Bearer sk_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  https://api.saveit.now/v1/bookmarks
```

## API Key Types

### Development Keys
- **Prefix:** `sk_test_`
- **Environment:** Sandbox/Development
- **Rate limits:** Lower limits for testing
- **Data:** Separate from production

### Production Keys  
- **Prefix:** `sk_live_`
- **Environment:** Live/Production
- **Rate limits:** Full rate limits
- **Data:** Your actual bookmarks

### Read-Only Keys
- **Prefix:** `sk_readonly_`
- **Permissions:** GET requests only
- **Use case:** Analytics, reporting, public displays

## Permissions and Scopes

When generating API keys, you can set specific permissions:

### Bookmark Permissions
- `bookmarks:read` - View bookmarks and collections
- `bookmarks:write` - Create and update bookmarks
- `bookmarks:delete` - Delete bookmarks

### Collection Permissions
- `collections:read` - View collections
- `collections:write` - Create and update collections
- `collections:delete` - Delete collections

### Account Permissions
- `account:read` - View account information
- `account:write` - Update account settings

### Example Permission Sets

#### **Analytics Integration** (Read-Only)
```json
{
  "permissions": [
    "bookmarks:read",

  ]
}
```

#### **Full Access Integration**

## Rate Limiting

The SaveIt API implements rate limiting to ensure fair usage:

### Rate Limit Headers
Every response includes rate limit information:

```http
HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1609459200
X-RateLimit-Reset-After: 3600
```

### Rate Limit Tiers

| Plan | Requests/Hour | Burst Limit |
|------|--------------|-------------|
| Free | 100 | 10 |
| Pro | 1,000 | 50 |
| Team | 5,000 | 100 |
| Enterprise | 50,000 | 500 |

### Handling Rate Limits

```javascript
async function makeAPIRequest(url, options) {
  const response = await fetch(url, options);
  
  if (response.status === 429) {
    const resetAfter = response.headers.get('X-RateLimit-Reset-After');
    console.log(`Rate limited. Retry after ${resetAfter} seconds`);
    
    // Wait and retry
    await new Promise(resolve => setTimeout(resolve, resetAfter * 1000));
    return makeAPIRequest(url, options);
  }
  
  return response;
}
```

## Error Handling

### Authentication Errors

#### Invalid API Key
```json
{
  "error": {
    "code": "invalid_api_key",
    "message": "The provided API key is invalid or has been revoked"
  }
}
```

#### Insufficient Permissions
```json
{
  "error": {
    "code": "insufficient_permissions",
    "message": "API key does not have permission to perform this action",
    "required_permission": "bookmarks:write"
  }
}
```

#### Expired Key
```json
{
  "error": {
    "code": "api_key_expired", 
    "message": "API key has expired. Please generate a new one"
  }
}
```

### Error Handling Best Practices

```javascript
async function handleAPIResponse(response) {
  if (!response.ok) {
    const error = await response.json();
    
    switch (error.error.code) {
      case 'invalid_api_key':
        throw new Error('Invalid API key. Please check your credentials.');
        
      case 'insufficient_permissions':
        throw new Error(`Missing permission: ${error.required_permission}`);
        
      case 'rate_limit_exceeded':
        const retryAfter = response.headers.get('Retry-After');
        throw new Error(`Rate limited. Retry after ${retryAfter} seconds.`);
        
      default:
        throw new Error(`API Error: ${error.error.message}`);
    }
  }
  
  return response.json();
}
```

## Security Best Practices

### ✅ Do This:
- **Rotate keys regularly** (every 90 days)
- **Use environment variables** for API keys
- **Implement exponential backoff** for rate limits
- **Use HTTPS only** for all API requests
- **Validate responses** before processing data
- **Log errors** but not sensitive data

### ❌ Don't Do This:
- **Hardcode API keys** in source code
- **Share keys** between environments
- **Ignore rate limits** - implement proper handling
- **Log full API responses** that might contain sensitive data
- **Use the same key** for multiple applications

### Key Rotation

```bash
# 1. Generate new key
NEW_KEY=$(curl -X POST https://api.saveit.now/v1/auth/keys \
  -H "Authorization: Bearer $CURRENT_KEY" \
  -d '{"name": "rotated-key-2024-01"}')

# 2. Update your application config
export SAVEIT_API_KEY=$NEW_KEY

# 3. Revoke old key after confirming new one works
curl -X DELETE https://api.saveit.now/v1/auth/keys/old_key_id \
  -H "Authorization: Bearer $NEW_KEY"
```
